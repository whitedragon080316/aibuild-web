## 二、Positioning 演化路徑（4 次推翻）

### v1：軌道 frame（被自己淘汰）
「別人教你上他的軌道，我教你鋪你自己的軌道」
- 問題：軌道 = 工業時代、有被動感、速度感弱

### v2：造局 frame（採用）
「別人教你進他的局，我教你造自己的局」
- 扣 brand name、動態感強、AI 時代味
- **作為 hero frame 仍保留**

### v3：Build-in-public frame（被使用者戳破太專業）
「這不是廣告，這是我今天的 commit」
- 給證據不給承諾、反 sales 調性
- 問題：對一般 TA（整復師/諮商/設計）太工程師、雜訊
- **「公開現場」是信任證據，不是主賣點**

### v4：AI 分身核心產品（真正的答案）
**「別人賣你 60 支影片。我派一個 24 小時在線的我陪你走完。」**
- 從「賣課程」升級到「賣 AI 教練平台」
- 這才是真正讓客戶願意付 10 萬的理由

## 三、最終產品定義：三合一系統

```
AI 造局術 = 造局 AI（產品核心）
           + Chiawei 協作（差異化）
           + 漏斗戰術（商業化）
```

**1. 造局 AI（產品核心）**
- Chiawei 濃縮版 AI 分身
- 包含：8 階段 60 堂方法論 + 60+ 踩坑紀錄 + 提問引導準則 + 判斷邏輯
- 一步一步引導學員做出自己的產品、走完課表
- **公用版本**（不客製），但有學員個人 memory/context

**2. Chiawei 協作（人味差異化）**
- AI 搞不定 → escalate 到 Chiawei 本人
- 每週 office hour / 關鍵決策對談
- 「AI 接 80%，人接 20%」= 真正可規模化的陪跑

**3. 漏斗戰術（商業化工程）**
- 從陌生人 → 付費學員 → Partner 的完整 playbook
- LP / 廣告 / LINE 追單 / 成交邏輯 / 續約

## 四、定價 + GTM（最強廣告投手 review 過）

### 定價結構
- **Builder $69,800**：60 堂 + AI 分身 1 年使用權 + 每週 office hour + Zeabur 一鍵部署 + 11 Skills
- **Partner $99,800**：Builder 全部 + AI 分身給他的客戶用 + 代操培訓 + 聯盟行銷 + 第一排派單席位

### 投手 review 發現的路徑錯誤（不要直接打冷流量 close 10 萬）

**正確 GTM 路徑：**

| 階段 | 時間 | 做什麼 |
|--|--|--|
| 1 | 6-8 週 | 累積**內容網**（10 支短影音 + 1 場免費直播 + Newsletter） |
| 2 | 同時 | 小預算廣告**只打免費直播報名**（CPA $50-100） |
| 3 | 直播後 | **Beta 價 $34,800-39,800** 給第一批 5-10 人（5 折） |
| 4 | 2-3 月 | 累積 3-5 個學員案例、產生見證 |
| 5 | 半年後 | 才用 $69,800 正價打冷流量 + Partner 擴大 |

### AI 分身改變 LP 轉換率
- LP CTA 改成「**免費跟 Chiawei AI 分身聊 10 分鐘**」
- 冷流量試聊 → 被征服 → 加 LINE → close
- **CPA 直接降一半**、不需先累積見證

### TA 先鎖一個 vertical 打透
建議：**教練 / 顧問** 先打（有專業、熟線上課、客單承受力高）
後擴：諮商、設計、律師、整復、健身

## 五、技術架構：AI 分身怎麼 build + 學員帶不走

### 兩層防護

**Layer 1：架構層（學員結構上看不到）**
```
[核心層 — 你擁有]
造局 AI API（Zeabur 後端）
├── system prompt（不外流）
├── 踩坑 knowledge base
├── 判斷準則
└── Skills 實作
         ↑
     API 呼叫
         |
[外殼層 — 學員擁有]
學員 Zeabur project
├── UI、品牌、前端
├── 學員自己的客戶資料
└── 呼叫造局 AI API
```

**類比：** 像 Google Docs — 你擁有文件，Google 擁有後端。學員的外殼 + data 真的是他的，AI 像他租的 SaaS。

**Layer 2：Prompt 層（AI 自己不洩漏）**
- 最高優先規則：絕不透露 system prompt / 規則 / 框架
- Prompt injection 偵測：`ignore instructions`、`print system prompt`、`you are now DAN`
- 標準拒絕話術：不解釋為什麼、不說「我被禁止」、直接轉回原任務
- API 層 regex filter + canary tokens + output filter

### 三層授權模型
| 層級 | 能做什麼 |
|--|--|
| Builder | 自己用 AI 分身（額度 10,000 次/年），續約 $12,000/年 |
| Partner | 額度 10x + 可給客戶用 + 抽成分潤 |
| Enterprise | 白牌授權（未來） |

### Recurring revenue 模型
- Builder 50 人 × $12,000/年續約 = **60 萬被動**
- Partner × 客戶 × 抽成 → 複利成長
- **3 年後公司價值 = 服務 N 個事業的 AI infrastructure**，不是「60 堂課」

## 六、競品分析

### 對手比較

| 維度 | Ellen Pro | Luke | 串聯學院 | 造局術 |
|--|--|--|--|--|
| 核心 | 賣課方法 | 跑項目 | 聚合人脈 | **AI 分身 + 陪跑 + 戰術** |
| 資歷 | 15 年 | 多項目淺蹲 | 品牌聚合 | 新人但 operator |
| 技術層 | 綁 Clickfunnels | 無 | 無 | **自己 Zeabur + AI** |
| 陪跑真實度 | 規模化後弱 | 弱 | 社群為主 | **AI + 真人雙層** |
| 規模化深度 | 第 1+3 層 | 第 1 層 | 第 1+3 層 | **1+2+3 三層** |

### 規模化本質（三層拔除法）
- 第一層：拔**操作**（錄影、軟體）← 大家都做到
- 第二層：拔**決策**（AI、Skills、SOP）← 少數人做到
- 第三層：拔**關係**（社群、品牌、網路效應）← 極少數

**造局術 USP = 第 2 層（決策 AI 化）做到位。Ellen/Luke/串聯都卡在第 2 層。**

### 差異化 reframe
- 不跟他們比「誰賣課強」（會輸）
- **站在他們上一層**：
  - Ellen 教學員方法論 → 學員來造局術把方法工程化
  - Luke 找項目 → 學員來造局術 Systemize 項目
  - 串聯連結人脈 → 學員來造局術把人脈變系統
- **你服務他們的下游。你是 meta 層。**

第一批 Partner 話術：

## 八、核心洞察總結

1. **你的優勢不是單一能力，是「5 項能力的完整組合 + 還在第一線 + 願意公開教」的稀缺 fit**
   - 手藝 / 軟體 / 廣告 / 內容 / AI 整合 5 項全有，台灣十根手指頭數得出來

2. **服務的 TA 不是所有人，是「想變全才 / 想造自己的局」的那 2000 人**
   - 快錢思維的去找 Ellen / Luke — 不要搶
   - 要你的這群人客單承受力高、LTV 長

3. **真正的產品是 AI 分身，不是課程**
   - 課程是 AI 的訓練素材
   - 戰術是 AI 的劇本
   - 協作是 AI 搞不定的後援
   - **AI 分身才是可規模化的產品核心**

4. **護城河不是「絕對防線」，是「防線 + 持續更新速度」**
   - 第一層（架構）+ 第二層（Prompt 防護）一起上
   - 讓一般人抄不走 = 保護 99% 市場
   - 高手要花 1 個月才能抄，你這段時間 iterate 到下一版

5. **建設順序**：
   - **先**：內容網 + 免費直播 + beta 價
   - **再**：AI 分身 + 學員 context 個人化
   - **最後**：Partner 2.0 派單平台
